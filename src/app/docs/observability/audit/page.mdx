# Audit Events

OpenGate IAM emits structured **audit events** via **Apache Kafka** for every authentication and administrative action. Use them for security monitoring, compliance, and SIEM integration.

## Event Architecture

```
Auth / Admin Action
  → opengate-auth-service (or admin-api)
  → Kafka Topic: opengate.audit.events
  → opengate-notification-service (consumer)
  → Storage / SIEM / Alert System
```

## Event Structure

Every audit event is a JSON object:

```json
{
  "eventId": "evt-uuid-1234",
  "eventType": "USER_LOGIN_SUCCESS",
  "timestamp": "2026-03-08T14:30:00.000Z",
  "realm": "master",
  "userId": "user-uuid",
  "username": "john_doe",
  "clientId": "sample-app",
  "ipAddress": "192.168.1.10",
  "userAgent": "Mozilla/5.0 (Macintosh; Intel Mac OS X 10_15_7)...",
  "sessionId": "sess-uuid",
  "details": {
    "grantType": "authorization_code",
    "scopes": ["openid", "profile"],
    "mfaUsed": true
  }
}
```

## Audit Event Types

### Authentication Events

| Event Type | Trigger |
|---|---|
| `USER_LOGIN_SUCCESS` | Successful password + MFA login |
| `USER_LOGIN_FAILED` | Wrong password or locked account |
| `USER_LOGOUT` | User or admin initiated logout |
| `TOKEN_ISSUED` | New access/refresh token granted |
| `TOKEN_REFRESHED` | Token refreshed via refresh_token |
| `TOKEN_REVOKED` | Token or session revoked |
| `MFA_CHALLENGE_SENT` | OTP or TOTP challenge issued |
| `MFA_VERIFIED` | MFA code verified successfully |
| `MFA_FAILED` | Incorrect MFA code submitted |

### Admin Events

| Event Type | Trigger |
|---|---|
| `USER_CREATED` | Admin created a new user |
| `USER_UPDATED` | User profile or password changed |
| `USER_DELETED` | User deleted from realm |
| `ROLE_ASSIGNED` | Role assigned to user or group |
| `ROLE_REMOVED` | Role removed from user |
| `CLIENT_CREATED` | New OAuth2 client registered |
| `CLIENT_SECRET_REGENERATED` | Client secret rotated |
| `REALM_UPDATED` | Realm configuration changed |

## Kafka Topic Configuration

```yaml
# application.yml (auth-service / admin-api)
spring:
  kafka:
    bootstrap-servers: localhost:9092
    producer:
      key-serializer: org.apache.kafka.common.serialization.StringSerializer
      value-serializer: org.springframework.kafka.support.serializer.JsonSerializer
      properties:
        spring.json.add.type.headers: false

opengate:
  audit:
    topic: opengate.audit.events
    enabled: true
```

## Consuming Audit Events

```java
@Component
public class AuditEventConsumer {

  @KafkaListener(topics = "opengate.audit.events", groupId = "audit-logger")
  public void handle(AuditEvent event) {
    log.info("[AUDIT] {} | user={} | ip={} | realm={}",
      event.getEventType(),
      event.getUsername(),
      event.getIpAddress(),
      event.getRealm()
    );
    // Forward to SIEM, database, or alerting system
  }
}
```

## Query Audit Events via API

```bash
# Get recent audit events for a realm
GET /admin/realms/master/audit-events?limit=50&eventType=USER_LOGIN_FAILED
Authorization: Bearer <admin_token>

# Response
[
  {
    "eventId": "evt-uuid",
    "eventType": "USER_LOGIN_FAILED",
    "timestamp": "2026-03-08T14:00:00Z",
    "username": "unknown_user",
    "ipAddress": "203.0.113.42",
    "details": { "reason": "INVALID_CREDENTIALS" }
  }
]
```

## SIEM Integration

Audit events can be forwarded to any SIEM tool:

```yaml
# Example: forward to Elasticsearch via Kafka Connect
connectors:
  - name: audit-to-elasticsearch
    config:
      connector.class: io.confluent.connect.elasticsearch.ElasticsearchSinkConnector
      topics: opengate.audit.events
      connection.url: http://elasticsearch:9200
      type.name: audit_event
      key.ignore: true
```

## Retention

Configure Kafka topic retention to meet your compliance requirements:

```bash
# Set 90-day retention on audit topic
kafka-configs.sh --bootstrap-server localhost:9092 \
  --entity-type topics \
  --entity-name opengate.audit.events \
  --alter \
  --add-config retention.ms=7776000000
```
