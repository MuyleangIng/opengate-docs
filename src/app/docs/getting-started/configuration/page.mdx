## Realm Settings

Realms are the top-level tenants in OpenGate. Each realm has isolated users, clients, and roles. The `master` realm is seeded automatically on first startup.

| Setting | Default | Description |
|---|---|---|
| `tokenLifespanSeconds` | `300` | Access token TTL (5 minutes) |
| `refreshTokenLifespanSeconds` | `2592000` | Refresh token TTL (30 days) |
| `mfaRequired` | `false` | Force MFA for all logins |
| `loginTheme` | `default` | UI theme for the login page |
| `passwordPolicy` | `null` | JSON password constraints |
| `smtpSettings` | `null` | JSON SMTP configuration |

## Auth Server & JWT Keys

By default, a 2048-bit RSA key pair is generated **in memory** on startup.

<Callout variant="warning" title="In-memory keys invalidate JWTs on restart">
  In-memory RSA keys are regenerated on every restart, invalidating all existing JWTs. Use HashiCorp Vault or a keystore file in production.
</Callout>

<CodeBlock language="yaml" filename="application.yml (auth-service, production)" code={`vault:
  uri: \${VAULT_URI:http://localhost:8200}
  token: \${VAULT_TOKEN:root}
  kv:
    enabled: true
    backend: secret
    default-context: opengate/jwt-keys`} />

## Password Policy

Configure per-realm via `passwordPolicy` (JSONB column):

<CodeBlock language="json" code={`{
  "minLength": 8,
  "requireUppercase": true,
  "requireNumbers": true,
  "requireSpecial": false,
  "maxAgeDays": 90,
  "historyCount": 5
}`} />

## SMTP Settings

Configure per-realm via `smtpSettings` (JSONB column):

<CodeBlock language="json" code={`{
  "host": "smtp.gmail.com",
  "port": 587,
  "username": "noreply@yourcompany.com",
  "password": "app-specific-password",
  "fromName": "OpenGate IAM",
  "ssl": false,
  "starttls": true
}`} />

<Callout variant="info" title="Per-realm SMTP">
  Each realm can have its own SMTP configuration, allowing different branding and sender addresses per tenant.
</Callout>
