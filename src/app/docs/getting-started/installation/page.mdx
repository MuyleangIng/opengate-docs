# Installation

## Local Development (Gradle)

Build all modules from the root:

<CodeBlock language="bash" code={`./gradlew build`} />

Run a single service:

<CodeBlock language="bash" code={`./gradlew :opengate-auth-service:bootRun`} />

## Docker Build

Build all service images:

<CodeBlock language="bash" code={`docker compose -f docker/docker-compose.services.yml build`} />

## Module Structure

```
opengate/
├── opengate-common/          # Shared library (exceptions, DTOs, events)
├── opengate-gateway/         # Port 8080 — API Gateway
├── opengate-auth-service/    # Port 8081 — OAuth2/OIDC
├── opengate-user-service/    # Port 8082 — User management
├── opengate-realm-service/   # Port 8083 — Multi-tenancy
├── opengate-rbac-service/    # Port 8084 — Roles & permissions
├── opengate-client-service/  # Port 8085 — OAuth clients
├── opengate-mfa-service/     # Port 8086 — MFA (TOTP, OTP)
├── opengate-session-service/ # Port 8087 — Session management
├── opengate-notification-service/ # Port 8088 — Email notifications
├── opengate-admin-api/       # Port 8089 — Admin REST API
├── opengate-console/         # Next.js 14 Admin UI (port 3000)
├── opengate-docs/            # Next.js 14 Documentation (port 3001)
└── docker/                   # Docker Compose files
```

## Flyway Migrations

Each service manages its own database schema via Flyway. Migrations run automatically on startup. Migration files are located at:

<CodeBlock language="text" code={`src/main/resources/db/migration/V1__create_<table>.sql`} />

<Callout variant="warning" title="Schema validation">
Services use `spring.jpa.hibernate.ddl-auto: validate`. The schema must match exactly — Flyway migrations must run before the application starts.
</Callout>
