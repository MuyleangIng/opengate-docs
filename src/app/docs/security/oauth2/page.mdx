# OAuth 2.1 / OIDC

OpenGate IAM implements **OAuth 2.1** and **OpenID Connect (OIDC)** via [Spring Authorization Server 1.3](https://spring.io/projects/spring-authorization-server). It is the recommended standard for securing modern web apps, APIs, and CLI tools.

## Supported Grant Types

| Grant Type | Use Case |
|---|---|
| `authorization_code` + PKCE | Browser apps, SPAs, mobile apps |
| `client_credentials` | Server-to-server / machine auth |
| `refresh_token` | Silent token renewal |
| `device_code` | CLI tools, smart TVs |

> **OAuth 2.1 removes** the implicit grant and requires PKCE for all authorization code flows.

## Authorization Code + PKCE Flow

This is the recommended flow for any app running in a browser.

```bash
# Step 1 — Generate code_verifier (random 64 bytes, base64url)
CODE_VERIFIER=$(openssl rand -base64 64 | tr '+/' '-_' | tr -d '=\n')

# Step 2 — Compute code_challenge (SHA-256 of verifier)
CODE_CHALLENGE=$(echo -n "$CODE_VERIFIER" | openssl dgst -sha256 -binary | base64 | tr '+/' '-_' | tr -d '=')

# Step 3 — Redirect user to authorization endpoint
GET http://localhost:9081/oauth2/authorize
  ?response_type=code
  &client_id=sample-app
  &redirect_uri=http://localhost:3003/callback
  &scope=openid profile email
  &code_challenge=$CODE_CHALLENGE
  &code_challenge_method=S256
  &state=<random_state>

# Step 4 — Exchange code for tokens
POST http://localhost:9081/oauth2/token
Content-Type: application/x-www-form-urlencoded

grant_type=authorization_code
&code=<auth_code>
&redirect_uri=http://localhost:3003/callback
&client_id=sample-app
&code_verifier=$CODE_VERIFIER
```

## OIDC Endpoints

| Endpoint | URL |
|---|---|
| Discovery | `http://localhost:9081/.well-known/openid-configuration` |
| Authorization | `http://localhost:9081/oauth2/authorize` |
| Token | `http://localhost:9081/oauth2/token` |
| JWKS | `http://localhost:9081/oauth2/jwks` |
| Userinfo | `http://localhost:9081/userinfo` |
| Revoke | `http://localhost:9081/oauth2/revoke` |

## JWT Access Token Structure

```json
{
  "sub": "user-uuid",
  "iss": "http://localhost:9081",
  "aud": ["sample-app"],
  "exp": 1700000000,
  "iat": 1699996400,
  "scope": "openid profile email",
  "roles": ["ROLE_USER", "ROLE_ADMIN"],
  "realm": "master",
  "email": "user@example.com"
}
```

Tokens are signed with **RSA-2048** keys. Verify using the JWKS endpoint.

## Registering a Public Client (PKCE)

```yaml
# application.yml excerpt for opengate-auth-service
clients:
  - client-id: my-spa
    client-authentication-methods:
      - none          # public client — no secret
    authorization-grant-types:
      - authorization_code
      - refresh_token
    redirect-uris:
      - http://localhost:3000/callback
    scopes:
      - openid
      - profile
      - email
    require-proof-key: true   # enforce PKCE
```

## Client Credentials (Server-to-Server)

```bash
curl -X POST http://localhost:9081/oauth2/token \
  -u "backend-service:secret" \
  -d "grant_type=client_credentials&scope=read write"
```

## Token Validation (Spring Boot Resource Server)

```java
@Configuration
@EnableWebSecurity
public class ResourceServerConfig {

  @Bean
  public SecurityFilterChain filterChain(HttpSecurity http) throws Exception {
    http
      .authorizeHttpRequests(auth -> auth
        .requestMatchers("/public/**").permitAll()
        .anyRequest().authenticated()
      )
      .oauth2ResourceServer(oauth2 -> oauth2
        .jwt(jwt -> jwt.jwkSetUri("http://localhost:9081/oauth2/jwks"))
      );
    return http.build();
  }
}
```
