# Multi-Factor Authentication (MFA)

OpenGate IAM provides MFA via the dedicated **`opengate-mfa-service`** (`:9086`). It supports TOTP, Email OTP, SMS OTP, and backup codes.

## Supported MFA Methods

| Method | Description | Recommended |
|---|---|---|
| **TOTP** | Time-based OTP — Google Authenticator, Authy | ✅ Yes |
| **Email OTP** | 6-digit code sent to user's email | ✅ Yes |
| **SMS OTP** | 6-digit code sent via SMS | ⚠️ Less secure |
| **Backup Codes** | One-time recovery codes | ✅ Always enable |

## TOTP Setup Flow

```
1. User enables MFA in account settings
2. Server generates a TOTP secret (Base32)
3. Server returns QR code URI  →  user scans with Authenticator app
4. User submits a verification code to confirm enrollment
5. MFA is now active on the account
```

## Enroll TOTP via API

```bash
# Step 1 — Start TOTP enrollment
POST /mfa/totp/enroll
Authorization: Bearer <access_token>

# Response
{
  "secret": "JBSWY3DPEHPK3PXP",
  "qrCodeUri": "otpauth://totp/OpenGate:user@example.com?secret=JBSWY3DPEHPK3PXP&issuer=OpenGate",
  "enrollmentId": "enroll-uuid"
}

# Step 2 — Confirm enrollment with a 6-digit code
POST /mfa/totp/confirm
Authorization: Bearer <access_token>
Content-Type: application/json

{
  "enrollmentId": "enroll-uuid",
  "code": "123456"
}
```

## Email OTP

```bash
# Trigger email OTP challenge
POST /mfa/email/challenge
Authorization: Bearer <access_token>

# Verify the code
POST /mfa/email/verify
Authorization: Bearer <access_token>
Content-Type: application/json

{
  "code": "847291"
}
```

## Backup Codes

```bash
# Generate 10 one-time backup codes
POST /mfa/backup-codes/generate
Authorization: Bearer <access_token>

# Response
{
  "codes": [
    "AAAA-BBBB", "CCCC-DDDD", "EEEE-FFFF",
    "GGGG-HHHH", "IIII-JJJJ", "KKKK-LLLL",
    "MMMM-NNNN", "OOOO-PPPP", "QQQQ-RRRR",
    "SSSS-TTTT"
  ]
}
```

> Store backup codes securely. Each code can only be used **once**.

## MFA Enforcement per Realm

You can enforce MFA for all users in a realm via the Admin API:

```bash
PATCH /admin/realms/master
Authorization: Bearer <admin_token>
Content-Type: application/json

{
  "mfaRequired": true,
  "mfaMethods": ["TOTP", "EMAIL_OTP"]
}
```

## MFA in the Login Flow

When MFA is enabled, the auth flow adds a second step after password verification:

```
Login (username + password)
  → Password verified ✅
  → MFA challenge issued
  → User submits OTP code
  → MFA verified ✅
  → Authorization code returned
```
