# Multi-Factor Authentication

OpenGate IAM provides MFA via the dedicated **`opengate-mfa-service`** (port `8086`). It supports TOTP, Email OTP, and backup codes.

## Supported Methods

| Method | Security Level | Description |
|---|---|---|
| **TOTP** | 🔒 High | Time-based OTP — Google Authenticator, Authy, 1Password |
| **Email OTP** | 🔒 Medium | 6-digit code delivered to verified email |
| **Backup Codes** | 🔒 Recovery | 10 one-time recovery codes — always generate these |

## MFA Login Flow

When `mfaRequired: true` is set on a realm:

```
  Client                Auth Service              MFA Service
    │                        │                         │
    │ 1. POST credentials    │                         │
    │───────────────────────►│                         │
    │                        │ 2. Password valid       │
    │ 3. { status:           │                         │
    │      "MFA_REQUIRED",   │                         │
    │      mfa_token }       │                         │
    │◄───────────────────────│                         │
    │                        │                         │
    │ 4. POST /api/mfa/      │                         │
    │    totp/verify         │                         │
    │    { code, mfa_token } │────────────────────────►│
    │                        │                         │ 5. Verify TOTP
    │                        │◄────────────────────────│
    │                        │ 6. Issue access_token   │
    │◄───────────────────────│                         │
```

## TOTP Enrollment

<CodeBlock language="bash" code={`# Step 1 — Initiate enrollment
curl -X POST \\
  -H "Authorization: Bearer $TOKEN" \\
  http://localhost:8080/api/mfa/totp/enroll`} />

**Response:**

<CodeBlock language="json" code={`{
  "secret": "JBSWY3DPEHPK3PXP",
  "qrCodeUri": "otpauth://totp/OpenGate:alice@example.com?secret=JBSWY3DPEHPK3PXP&issuer=OpenGate",
  "enrollmentId": "enroll-uuid-123"
}`} />

<CodeBlock language="bash" code={`# Step 2 — Confirm with first 6-digit code from authenticator app
curl -X POST \\
  -H "Authorization: Bearer $TOKEN" \\
  -H "Content-Type: application/json" \\
  -d '{ "enrollmentId": "enroll-uuid-123", "code": "123456" }' \\
  http://localhost:8080/api/mfa/totp/confirm`} />

<Callout variant="info" title="TOTP implementation">
  OpenGate uses a custom **HMAC-SHA1 RFC 6238** implementation with ±1 time-window tolerance (30-second windows). Compatible with Google Authenticator, Authy, and any RFC 6238-compliant app.
</Callout>

## Email OTP

<CodeBlock language="bash" code={`# Trigger email OTP (sends 6-digit code to verified email)
curl -X POST \\
  -H "Authorization: Bearer $TOKEN" \\
  http://localhost:8080/api/mfa/email-otp/send

# Verify the code (valid for 10 minutes)
curl -X POST \\
  -H "Authorization: Bearer $TOKEN" \\
  -H "Content-Type: application/json" \\
  -d '{ "code": "847291" }' \\
  http://localhost:8080/api/mfa/email-otp/verify`} />

## Backup Codes

<CodeBlock language="bash" code={`# Generate 10 one-time backup codes
curl -X POST \\
  -H "Authorization: Bearer $TOKEN" \\
  http://localhost:8080/api/mfa/backup-codes/generate`} />

<CodeBlock language="json" code={`{
  "codes": [
    "AAAA-BBBB", "CCCC-DDDD", "EEEE-FFFF",
    "GGGG-HHHH", "IIII-JJJJ", "KKKK-LLLL",
    "MMMM-NNNN", "OOOO-PPPP", "QQQQ-RRRR",
    "SSSS-TTTT"
  ]
}`} />

<Callout variant="warning" title="Each backup code can only be used once">
  Store backup codes in a password manager. Once all 10 are used you must regenerate. If lost, an admin can disable MFA for the account.
</Callout>

## Enforce MFA per Realm

<CodeBlock language="bash" code={`curl -X PUT \\
  -H "Authorization: Bearer $ADMIN_TOKEN" \\
  -H "Content-Type: application/json" \\
  -d '{ "mfaRequired": true }' \\
  http://localhost:8080/api/realms/my-realm`} />

When enabled, every user in the realm must complete MFA before receiving tokens.
